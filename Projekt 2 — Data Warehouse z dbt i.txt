Projekt 2 — Data Warehouse z dbt i Airflow
Flight Intelligence: Military Activity Anomaly Detector
Repo name: dbt-flight-intelligence-dwh
Czas realizacji: Miesiąc 2 (≈ 4 tygodnie)
Stack: Python · dbt · BigQuery · Apache Airflow · Docker · NetworkX · OpenSky Network API

Cel projektu
Zbudować hurtownię danych opartą na publicznych danych lotniczych (OpenSky Network API), która:

Przechowuje dane w schemacie gwiazdkowym (star schema) w BigQuery
Transformuje dane przez warstwy dbt (staging → intermediate → marts)
Implementuje dwa zaawansowane algorytmy analityczne:

Military Activity Anomaly Detector — wykrywa wzrosty natężenia lotów wojskowych ponad statystyczną normę (potencjalne ćwiczenia / działania zbrojne)
Airport Network Graph Analysis — analiza grafu połączeń lotniczych z PageRank i betweenness centrality




Źródło danych
OpenSky Network API (https://opensky-network.org)

Darmowe, publiczne API z danymi ADS-B
Pokrycie: globalny, najlepsza jakość dla Europy
Dane historyczne: do 30 dni wstecz (darmowy tier), bezterminowo dla zarejestrowanych kont
Endpoint: /flights/all?begin=<ts>&end=<ts> — wszystkie loty w danym oknie czasowym
Endpoint: /states/all — aktualny stan przestrzeni powietrznej (real-time)

Identyfikacja samolotów wojskowych:

Każdy samolot ma kod icao24 (6-znakowy hex przydzielony przez ICAO)
Zakresy ICAO24 dla wojska są publiczne — np.:

3B0000–3BFFFF → Niemcy (Bundeswehr)
4B0000–4BFFFF → Szwajcaria
448000–44FFFF → Polska (LOT wojskowy + Air Force)
ADF000–ADFFFF → US Air Force


Skrzyżowanie z bazą danych Mictronics ICAO Military DB lub OpenSky własna baza

Algorytm 1: Military Activity Anomaly Detector
Cel
Wykryć dni/tygodnie, w których natężenie lotów wojskowych znacząco przekracza statystyczną normę dla danego regionu — potencjalny sygnał ćwiczeń wojskowych lub ruchów zbrojnych.
Metodologia
Krok 1 — Baseline (dbt model):
Oblicz dla każdego kraju/regionu i dla każdego dnia tygodnia historyczną medianę liczby lotów wojskowych:
sql-- int_daily_military_counts.sql
SELECT
    date_id,
    EXTRACT(DAYOFWEEK FROM date_id) AS day_of_week,
    country_code,
    COUNT(*) AS military_flight_count
FROM int_military_flights
GROUP BY 1, 2, 3
Krok 2 — Z-score (dbt Python model):
python# mart_military_anomaly_scores.py
import pandas as pd
from scipy import stats

def model(dbt, session):
    df = dbt.ref("int_daily_military_counts").to_pandas()

    # Oblicz Z-score per kraj per dzień tygodnia
    df["baseline_mean"] = df.groupby(["country_code", "day_of_week"])["military_flight_count"].transform("mean")
    df["baseline_std"] = df.groupby(["country_code", "day_of_week"])["military_flight_count"].transform("std")
    df["z_score"] = (df["military_flight_count"] - df["baseline_mean"]) / df["baseline_std"]

    # Klasyfikacja anomalii
    df["anomaly_level"] = pd.cut(
        df["z_score"],
        bins=[-float("inf"), 1.5, 2.5, 3.5, float("inf")],
        labels=["NORMAL", "ELEVATED", "HIGH", "CRITICAL"]
    )

    return df[["date_id", "country_code", "military_flight_count",
               "baseline_mean", "z_score", "anomaly_level"]]
Krok 3 — Kontekstualizacja:

Połączenie z kalendarzem wydarzeń NATO (publiczne dane) — czy anomalia pokrywa się ze znalnym ćwiczeniem?
Rolling window 7-dniowy dla wygładzenia szumów

Output: Tabela mart_military_anomaly_scores z flagami i poziomami alertu (NORMAL / ELEVATED / HIGH / CRITICAL)

Algorytm 2: Airport Network Graph Analysis
Cel
Modelowanie sieci połączeń lotniczych jako graf skierowany — identyfikacja kluczowych hubów przy użyciu miar centralności (PageRank, betweenness centrality).
Metodologia
python# mart_pagerank_airports.py
import pandas as pd
import networkx as nx

def model(dbt, session):
    edges = dbt.ref("int_airport_connections").to_pandas()
    # edges: origin_airport_id, destination_airport_id, flight_count (waga)

    # Zbuduj skierowany ważony graf
    G = nx.DiGraph()
    for _, row in edges.iterrows():
        G.add_edge(
            row["origin_airport_id"],
            row["destination_airport_id"],
            weight=row["flight_count"]
        )

    # Metryki centralności
    pagerank = nx.pagerank(G, weight="weight")
    in_degree = dict(G.in_degree(weight="weight"))
    out_degree = dict(G.out_degree(weight="weight"))
    betweenness = nx.betweenness_centrality(G, weight="weight")

    results = pd.DataFrame({
        "airport_id": list(pagerank.keys()),
        "pagerank_score": list(pagerank.values()),
        "in_degree_weighted": [in_degree.get(n, 0) for n in pagerank.keys()],
        "out_degree_weighted": [out_degree.get(n, 0) for n in pagerank.keys()],
        "betweenness_centrality": [betweenness.get(n, 0) for n in pagerank.keys()],
        "hub_rank": range(1, len(pagerank) + 1)  # sorted by pagerank
    })

    return results.sort_values("pagerank_score", ascending=False)
Output: Tabela mart_pagerank_airports — ranking lotnisk z metrykami grafowymi
